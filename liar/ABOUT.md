# Liar Module

이 문서는 `liar/` 모듈 설명서다. 저장소 루트 설치, `.env` 위치, 공통 `npm` 명령, PM2 운영은 [README.md](../README.md) 를 기준으로 본다.

`liar/` 는 `Discord Game Bot` 안에서 Discord 채널 기반 `라이어게임`을 담당한다. 웹 대시보드는 쓰지 않고, 생성된 채널 안에서 버튼, 일반 메시지, `!투표` prefix, `/제시어` ephemeral 응답, `/liar stats` 전적 조회만으로 한 판을 운영한다.

## 현재 입력 체계

- slash command: `/liar create`, `/liar stats`, `/제시어`
- 버튼: `참가`, `나가기`, `시작`, `투표 시작`, `강제 집계`, `종료`
- 로비 select: `카테고리`, `규칙 모드`
- prefix: `!투표 @대상`
- 일반 메시지
  - 설명 단계의 한 줄 설명
  - 라이어 추리 단계의 최종 정답 입력

## 규칙 모드

- `모드A`: 라이어는 자신이 라이어임을 알고 제시어를 받지 않는다
- `모드B`: 라이어는 자신이 라이어인지 모른 채 혼자 다른 제시어를 받는다
- `모드B` 는 선택된 카테고리에 서로 다른 제시어가 최소 2개 이상 있어야 시작할 수 있다

## 현재 범위

- `4~8인`
- 서버당 동시 게임 `1개`
- 제시어 카테고리 기반 출제
- 제시어 소스: `liar/resource/categories.json`
- 길드별 제시어 팩 소스: `liar/resource/guild-categories.json`
- 로비에서 `모드A` 또는 `모드B` 선택 가능
- `/제시어` 기반 개인 정보 전달
- `/liar stats` 기반 Discord 전적 조회
- 채널 안에서 로비, 설명, 토론, 투표, 추리, 종료까지 진행
- 상태 저장은 `in-memory`

## 안정화 / UX 상태

- 로비 `10분` 무활동 시 자동 종료
- 설명 `45초`, 투표 `45초`, 추리 `30초`
- 토론은 인원수 기반 자동 타이머로 진행 후 자동 투표 시작
- 단계 종료 전 경고 메시지 발송
- 상태 메시지에 단계, 진행도, 참가자, 남은 시간을 함께 표시
- 설명/투표/추리 단계에서 허용되지 않은 참가자 메시지는 정리되거나 안내 메시지로 되돌려진다
- 게임 시작 뒤 참가자가 서버에서 나가면 해당 판은 즉시 `취소 종료` 된다
- 같은 서버/같은 카테고리에서는 제시어 풀이 한 번 소진될 때까지 최근 사용 단어를 재사용하지 않는다
- `DATABASE_URL` 이 있으면 종료된 라이어게임이 Postgres shared business DB 에 기록된다
- `DATABASE_URL` 이 없으면 라이어 전적도 로컬 파일 `mafia/runtime-data/game-stats.json` 에 함께 기록된다
- `/liar stats` 는 완료/취소 판수, 역할별 전적, 연승, 카테고리별 전적, 최근 경기를 요약해 보여준다

## 운영 메모

- 이 모듈은 Discord `Message Content Intent` 가 켜져 있어야 `!투표` 와 일반 메시지 입력을 처리할 수 있다.
- 참가자 잡음 메시지를 자동 정리하려면 봇에 `메시지 관리` 권한이 있는 편이 좋다. 권한이 없으면 안내 메시지만 보낸다.
- 재시작 시 진행 중 게임은 복구되지 않는다.
- 재접속 복구는 현재 구현하지 않는다. Discord 채널 재진입만으로 플레이는 계속 볼 수 있고, 우선순위는 낮다.

## 길드별 팩 메모

- `liar/resource/guild-categories.json` 의 `mode` 는 `extend` 또는 `replace` 다.
- `extend`: 기본 카테고리를 유지한 채 길드 전용 카테고리를 추가하거나 같은 id 를 덮어쓴다.
- `replace`: 해당 길드에서 기본 카테고리 대신 길드 전용 목록만 사용한다.
