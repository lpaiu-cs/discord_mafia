# 라이어게임 제작 계획서

## 목표

- 현재 Discord 게임봇에 `마피아`와 별개로 `라이어게임`을 추가한다.
- 라이어게임은 `웹 대시보드 없이 Discord 채팅/버튼/선택 메뉴/DM`만으로 진행한다.
- 봇은 장기적으로 `게임 선택형 플랫폼`으로 확장한다.

## 제품 방향

- 마피아: `Discord 로비 + Web Dashboard` 기반
- 라이어게임: `Discord 채널 + DM` 기반
- 두 게임은 같은 봇 안에서 공존하되, 실행 흐름과 UI 매체는 분리한다.

## 라이어게임 운영 구조 제안

### 공개 채널

- 방 생성
- 참가 모집
- 게임 시작
- 순서 안내
- 토론 진행
- 최종 투표 결과 공개

### DM

- 공통 제시어 전달
- 라이어 여부 전달
- 개인 역할 안내

## 권장 게임 플로우

1. `/liar create`
2. 참가자 모집
3. 방장이 카테고리/주제 선택
4. 봇이 DM으로 제시어 또는 라이어 여부 전달
5. 순서대로 한 문장씩 설명
6. 자유 토론
7. 투표로 라이어 지목
8. 라이어가 제시어 추리 기회 획득
9. 승패 판정 및 결과 공개

## 1차 구현 범위

- 4~8인 고정
- 라이어 1명
- 제시어 카테고리 기반 출제
- DM 기반 비공개 정보 전달
- Discord 버튼/선택 메뉴 기반 진행
- 단판 승패 판정

## 2차 구현 범위

- 다중 라이어/스파이 변형 룰
- 라운드제 점수판
- 서버별 제시어 팩
- 금지어/중복 제시어 방지
- 플레이 기록 저장

## 권장 디렉토리 구조

```text
liar/
  ABOUT.md
  PLAN.md
  src/
    index.ts
    commands.ts
    engine/
      model.ts
      game.ts
      rules.ts
      phases.ts
    discord/
      presenter.ts
      controls.ts
      dm.ts
    content/
      prompts.ts
      categories.ts
  tests/
  resource/
```

## 기술 설계 포인트

- 상태 저장은 초기에는 메모리 기반으로 충분하다.
- 마피아와 달리 웹 세션, 웹 라우팅, 웹 자산 빌드는 필요 없다.
- 게임 종료 후 전적을 넣고 싶다면 이후 `shared business db` 계층에 연결한다.
- 제시어 데이터는 코드 상수로 시작하고, 이후 DB 또는 리소스 파일로 확장한다.

## 공통 플랫폼과의 접점

- 루트 Discord 게임봇은 `mafia/`, `liar/` 같은 게임 모듈을 선택적으로 로드할 수 있어야 한다.
- 상위 Discord 명령 체계는 게임명 단위 네임스페이스를 허용해야 한다.
- 향후 `/game create mafia`, `/game create liar` 같은 통합 진입점도 검토할 수 있다.

## 구현 순서 제안

1. `liar/src` 기본 골격 생성
2. `/liar create/join/leave/start/end` 명령 추가
3. 메모리 기반 방/세션 상태 구현
4. DM 비공개 정보 전달 구현
5. 토론/투표/추리/승패 판정 구현
6. 테스트 작성
7. 전적 저장 여부 결정

## 리스크

- DM 차단 유저 처리
- 응답 지연 시 순서 진행 꼬임
- 제시어 유출 방지
- 중도 이탈 처리

## 성공 기준

- 마피아 웹 흐름과 충돌 없이 별도 명령으로 라이어게임을 열 수 있다.
- 라이어게임 한 판이 Discord 안에서 완결된다.
- 신규 게임 추가 시 루트 아래 `/<new-game>` 모듈 패턴을 그대로 반복할 수 있다.
