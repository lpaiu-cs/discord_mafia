# RULE.md

## 문서 상태

- 이 문서는 `liar/` 모듈의 단일 기준 규칙 문서다.
- 마지막 정리 기준일은 `2026-03-27` 이다.
- 현재 범위는 Discord 서버 안에서 돌아가는 `채널형 라이어게임` 1차 규칙이다.
- 이 문서는 `확정`, `복원`, `미확정`을 구분한다.
- 현재 라이어게임 규칙은 이 저장소 고유 설계이므로 `복원` 항목은 없다.

## 적용 범위

- 플랫폼: Discord
- 인원: `4~8인`
- 서버당 동시 게임 수: `1개`
- 진행 채널: `/liar create` 를 입력한 채널
- 웹 대시보드: 사용하지 않음
- 비공개 정보 전달: `/제시어` 의 ephemeral 응답만 사용

## 입력 수단

### 확정

- slash command
  - `/liar create`
  - `/liar stats`
  - `/제시어`
- 버튼
  - `참가`
  - `나가기`
  - `시작`
  - `투표 시작`
  - `강제 집계`
  - `종료`
- 로비 select
  - `카테고리`
  - `규칙 모드`
- prefix 입력
  - `!투표 @대상`
- 일반 채널 메시지
  - 설명 단계의 한 줄 설명
  - 라이어 추리 단계의 최종 정답 입력

### 미확정

- 향후 추가 prefix 명령을 늘릴지 여부
- 일반 메시지 대신 modal/select 를 더 도입할지 여부

## 게임 단위와 방 생성

### 확정

- 한 서버에서는 동시에 하나의 라이어게임만 진행할 수 있다.
- `/liar create` 를 실행한 사용자가 방장이 된다.
- 방장은 로비 상태 메시지에서 카테고리를 고르고 게임을 시작한다.
- 방장은 로비 상태 메시지에서 카테고리와 규칙 모드를 고른다.
- 방장이 로비에서 나가면 가장 먼저 들어온 다음 참가자가 새 방장이 된다.
- 마지막 참가자까지 나가면 게임은 종료 상태로 전환된다.

### 미확정

- 종료된 게임 상태 메시지를 자동 삭제할지 여부

## 참가 규칙

### 확정

- 방장은 로비 생성과 동시에 자동 참가한다.
- 참가와 나가기는 로비 단계에서만 가능하다.
- 게임이 시작된 뒤 참가자가 서버에서 나가면 해당 판은 즉시 `취소 종료` 된다.
- 시작 가능 인원은 `4명 이상 8명 이하` 다.
- 관전자는 채널에 남아 있을 수 있지만 게임 입력 권한은 없다.

### 미확정

- 시작 후 재접속 복구를 제공할지 여부

## 역할과 제시어

### 확정

- 라이어는 `1명` 이다.
- 규칙 모드는 `모드A`, `모드B` 두 가지다.
- `모드A`: 라이어는 자신이 라이어임을 알고 제시어를 받지 않는다.
- `모드B`: 라이어는 자신이 라이어인지 모른 채 혼자 다른 제시어를 받는다.
- `모드B` 를 쓰는 카테고리는 서로 다른 제시어가 최소 `2개 이상` 있어야 한다.
- 일반 참가자는 모두 같은 제시어를 받는다.
- 제시어 카테고리는 현재 `음식`, `동물`, `직업`, `장소`, `사물` 다섯 가지다.
- 제시어 데이터는 `liar/resource/categories.json` 을 기준으로 불러온다.
- `liar/resource/guild-categories.json` 에 길드 id 가 있으면 해당 길드의 카테고리 팩을 `extend` 또는 `replace` 방식으로 적용한다.
- 제시어는 선택된 카테고리 안에서 무작위로 정한다.
- 같은 서버/같은 카테고리에서는 단어 풀이 한 번 소진될 때까지 최근 사용 제시어를 다시 뽑지 않는다.
- 참가자는 게임 시작 후 `/제시어` 를 입력해 자기 정보만 확인한다.
- `모드A` 의 라이어는 `/제시어` 에서 카테고리만 받고 실제 단어는 받지 않는다.
- `모드B` 의 라이어는 `/제시어` 에서 혼자 다른 제시어를 받는다.

### 미확정

- 서버별 카테고리 팩 관리 UI
- 카테고리별 가중치 또는 난이도

## 진행 단계

### 확정

1. 로비
2. 제시어 확인
3. 설명 라운드
4. 자유 토론
5. 투표
6. 라이어 추리
7. 종료

## 로비 단계

### 확정

- 방장은 상태 메시지의 카테고리 선택 메뉴로 주제를 바꿀 수 있다.
- 방장은 상태 메시지의 규칙 모드 선택 메뉴로 `모드A` 와 `모드B` 를 바꿀 수 있다.
- 참가자는 버튼으로 들어오고 나간다.
- 방장은 `시작` 버튼으로 게임을 연다.

## 설명 라운드

### 확정

- 설명 순서는 무작위다.
- 현재 차례의 플레이어가 채널에 일반 메시지 한 줄을 입력하면 그 내용이 설명으로 채택된다.
- 설명 길이는 `120자 이하` 로 제한한다.
- 설명 제한 시간은 `45초` 다.
- 제한 시간 안에 설명이 제출되지 않으면 그 플레이어의 차례는 자동으로 건너뛴다.
- 현재 차례가 아닌 참가자의 일반 메시지는 삭제를 시도하고, 봇이 차례 안내 메시지를 보낸다.
- 모든 참가자가 한 번씩 설명하면 자동으로 자유 토론 단계로 넘어간다.

### 미확정

- 설명 메시지를 반드시 한 문장으로 강제할지 여부

## 자유 토론

### 확정

- 설명 라운드가 끝나면 자유 토론 단계가 된다.
- 토론은 생성된 채널에서 일반 메시지로 진행한다.
- 토론 종료는 방장의 `투표 시작` 버튼 또는 자동 시간 만료로 이뤄진다.
- 토론 시간은 `max(60초, min(120초, 참가자 수 x 15초))` 다.
- 시간이 끝나면 자동으로 투표 단계가 시작된다.

### 미확정

- 토론 로그를 자동 요약할지 여부

## 관전자 정책

### 확정

- 관전자 일반 메시지는 삭제하지 않고 그대로 둔다.
- 관전자의 일반 메시지는 게임 입력으로 처리하지 않는다.
- 관전자가 `!투표` 를 입력하면 투표 권한이 없다는 안내만 보낸다.

## 투표

### 확정

- 투표는 생성된 채널에서 `!투표 @대상` 형식으로 입력한다.
- 참가자 한 명당 한 번만 투표할 수 있다.
- 자기 자신에게 투표하는 것도 현재 구현에서는 허용한다.
- 투표 시간은 `45초` 다.
- 참가자가 `!투표 @대상` 이 아닌 일반 메시지를 보내면 삭제를 시도하고, 봇이 투표 형식을 다시 안내한다.
- 방장은 모든 표가 모이지 않아도 `강제 집계` 버튼으로 현재 표를 집계할 수 있다.
- 시간이 끝나면 현재까지 제출된 표로 자동 집계한다.
- 시간 만료 시 제출된 표가 하나도 없으면 라이어 승리다.
- 최다 득표자가 동률이면 라이어 승리다.
- 라이어가 아닌 사람이 최다 득표자가 되면 즉시 라이어 승리다.
- 라이어가 지목되면 추리 단계로 넘어간다.

### 미확정

- 자기 자신 투표를 금지할지 여부
- 기권을 허용할지 여부
- 동률 시 재투표를 둘지 여부

## 라이어 추리

### 확정

- 라이어가 지목되면 그 라이어는 채널에 일반 메시지로 최종 정답을 한 번 입력한다.
- 추리 길이는 `60자 이하` 로 제한한다.
- 추리 제한 시간은 `30초` 다.
- 라이어가 아닌 참가자의 일반 메시지는 삭제를 시도하고, 봇이 입력 권한이 없음을 안내한다.
- 추리가 실제 제시어와 같으면 라이어 승리다.
- 추리가 틀리면 시민팀 승리다.
- 제한 시간 안에 정답을 제출하지 못하면 시민팀 승리다.

### 미확정

- 라이어 추리 기회를 여러 번 줄지 여부

## 승패 판정

### 확정

- 라이어 미지목: 라이어 승리
- 투표 시간 만료 후 무투표: 라이어 승리
- 투표 동률: 라이어 승리
- 라이어 지목 후 정답 성공: 라이어 승리
- 라이어 지목 후 정답 실패: 시민팀 승리
- 라이어 지목 후 시간 초과: 시민팀 승리
- 방장 종료 또는 참가자 전원 이탈: 승자 없는 `취소 종료`

## 상태 메시지

### 확정

- 각 게임은 채널에 상태 메시지 `1개` 를 유지한다.
- 로비, 설명, 토론, 투표, 추리, 종료 단계에 따라 상태 메시지 내용과 버튼 구성이 달라진다.
- 상태 메시지에는 현재 단계, 참가자, 진행도, 남은 시간이 표시된다.
- 종료 상태에서는 새 조작 버튼을 제공하지 않는다.

## 운영 전제

### 확정

- 게임 상태는 메모리에만 저장한다.
- 따라서 프로세스 재시작 시 진행 중 라이어게임은 복구되지 않는다.
- 현재 구현은 Discord `Message Content Intent` 가 활성화되어야 투표 prefix 와 일반 메시지 설명 입력을 처리할 수 있다.
- 참가자 잡음 메시지를 자동 삭제하려면 Discord에서 봇에 `메시지 관리` 권한이 있어야 한다. 권한이 없어도 게임은 진행되지만 안내 메시지만 보낸다.
- 종료된 라이어게임 레코드는 shared business DB 가 켜져 있으면 별도 라이어 전적 테이블에 저장된다.
- 저장된 라이어 전적은 `/liar stats` 로 ephemeral 조회할 수 있다.
- `/liar stats` 는 완료/취소 판수, 역할별 전적, 연승, 카테고리별 전적, 최근 경기를 요약한다.
- 로비는 `10분` 동안 활동이 없으면 자동 종료된다.
- 각 단계 종료 `10초 전` 에 경고 메시지를 보낸다. 단, 로비는 종료 `1분 전` 에 경고한다.

### 미확정

- 재시작 후 진행 중 게임을 복구할지 여부
- `/liar stats` 에 타인 조회, 기간 필터, 상세 전적 보기까지 넣을지 여부

## 테스트 기준

### 확정

- 엔진 테스트는 시작, 제시어 조회, 설명 진행, 투표 결과, 동률 판정, 제시어 제외 로직을 검증한다.
- Discord 서비스 테스트는 일반 메시지 설명 처리, `!투표` prefix 집계, 비차례 메시지 통제, 참가자 이탈 종료, 규칙 모드 선택, `/liar stats` 응답을 검증한다.

## 구현 메모

### 확정

- 현재 구현은 `liar/src/engine/game.ts`, `liar/src/discord/service.ts` 를 기준으로 동작한다.
- 로비 생성 명령은 `/liar create`, 개인 정보 확인은 `/제시어`, 전적 조회는 `/liar stats` 로 고정한다.
- 투표 prefix 는 `!투표` 로 고정한다.

## 미확정 항목 요약

- 자기 자신 투표 허용 여부
- 동률 재투표 여부
- 서버별 제시어 팩 관리 UI
- 재접속 처리
- 라이어 전적 상세 필터
