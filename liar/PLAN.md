# 라이어게임 제작 계획서

## 목표

- 현재 Discord 게임봇에 `마피아`와 별개로 `라이어게임`을 추가한다.
- 라이어게임은 `웹 대시보드 없이 Discord 채팅/버튼/prefix/ephemeral 응답`만으로 진행한다.
- 봇은 장기적으로 `게임 선택형 플랫폼`으로 확장한다.

## 제품 방향

- 마피아: `Discord 로비 + Web Dashboard` 기반
- 라이어게임: `Discord 채널 + 최소 slash command + 버튼 + prefix` 기반
- 두 게임은 같은 봇 안에서 공존하되, 실행 흐름과 UI 매체는 분리한다.

## 라이어게임 운영 구조 제안

### 공개 채널

- 방 생성
- 참가 모집
- 게임 시작
- 순서 안내
- 토론 진행
- `!투표 @대상` 입력
- 최종 투표 결과 공개

### 개인 응답

- 참가 유저가 `/제시어` 를 입력하면 봇이 ephemeral message 로 응답
- `모드A`: 일반 유저에게는 공통 제시어 전달, 라이어에게는 라이어 여부만 전달
- `모드B`: 일반 유저에게는 공통 제시어 전달, 라이어에게는 혼자 다른 제시어 전달
- 게임 중 재조회가 필요하면 동일 명령으로 다시 확인 가능

## 권장 게임 플로우

1. `/liar create`
2. 버튼으로 참가 모집
3. 방장이 상태 메시지에서 카테고리와 규칙 모드 선택
4. 방장이 `시작` 버튼을 누름
5. 각 참가자가 `/제시어` 로 자기 정보를 확인
6. 순서대로 채널에 한 줄씩 설명
7. 자유 토론
8. 방장이 `투표 시작` 버튼을 누름
9. 참가자들이 `!투표 @대상` 으로 투표
10. 라이어가 지목되면 채널에 최종 정답 입력
11. 승패 판정 및 결과 공개

## 1차 구현 범위

- 4~8인 고정
- 서버당 동시 게임 1개
- 라이어 1명
- 규칙 모드 `모드A`, `모드B`
- 제시어 카테고리 기반 출제
- `liar/resource/categories.json` 기반 제시어 로딩
- 같은 서버/같은 카테고리 내 최근 제시어 중복 방지
- `/제시어` 기반 비공개 정보 전달
- Discord 버튼 + 일반 메시지 + `!투표` 기반 진행
- 단계별 타이머와 경고 메시지
- 게임 중 참가자 서버 이탈 시 즉시 취소 종료
- 설명/투표/추리 단계의 비허용 참가자 메시지 통제
- 관전자 일반 메시지 비개입 정책
- shared business DB 라이어 전적 저장
- `/liar stats` 기반 Discord 전적 조회
- 단판 승패 판정
- `liar/resource/guild-categories.json` 기반 길드별 카테고리 팩 override

## 2차 구현 범위

- 다중 라이어/스파이 변형 룰
- 라운드제 점수판
- 서버별 제시어 팩 관리 UI/DB
- 라이어 전적 상세 UI/필터
- 재접속 복구 또는 재참가 정책

## 기술 설계 포인트

- 상태 저장은 현재 메모리 기반으로 충분하다.
- 마피아와 달리 웹 세션, 웹 라우팅, 웹 자산 빌드는 필요 없다.
- DM 의존 없이 slash command interaction 만으로 비공개 정보 조회를 처리한다.
- slash command 는 최소화하고 게임 내 조작은 채널 메시지와 버튼으로 처리한다.
- 규칙 모드는 로비 상태 메시지에서 `모드A` 또는 `모드B` 로 전환한다.
- 게임 종료 후 전적은 현재 `shared business db` 계층에 저장하고, 기본 조회는 `/liar stats` 로 제공한다.
- 제시어 데이터는 현재 리소스 파일로 분리했고, 길드별 팩도 `guild-categories.json` 으로 override 할 수 있다. 이후 관리 UI 나 DB 로 확장한다.

## 공통 플랫폼과의 접점

- 루트 Discord 게임봇은 `mafia/`, `liar/` 같은 게임 모듈을 선택적으로 로드할 수 있어야 한다.
- 상위 Discord 명령 체계는 게임별 최소 진입 slash command 를 허용해야 한다.
- 향후 `/game create mafia`, `/game create liar` 같은 통합 진입점도 검토할 수 있다.

## 구현 순서 제안

1. `liar/src` 기본 골격 생성
2. `/liar create`, `/liar stats`, `/제시어` 명령 추가
3. 메모리 기반 서버 단위 방 상태 구현
4. 버튼 기반 로비/단계 제어 구현
5. 일반 메시지 설명과 `!투표` prefix 흐름 구현
6. 토론/추리/승패 판정 구현
7. 테스트 작성
8. 전적 저장/조회 고도화

## 리스크

- ephemeral 응답을 캡처하거나 외부로 공유하는 유저를 시스템적으로 막을 수 없음
- 자동 타이머가 느린 채널/모바일 환경에서 촉박하게 느껴질 수 있음
- `/제시어` 재조회 가능 횟수와 허용 시점 정책 필요
- `Message Content Intent` 의존
- 메시지 삭제 기반 잡음 정리는 `메시지 관리` 권한이 없으면 완전하지 않음
- `/liar stats` 는 텍스트 요약 위주라 상세 필터나 누적 분석은 아직 없다
- 길드별 팩은 현재 리소스 파일 편집 방식이라 Discord 안에서 바로 관리할 수는 없다

## 성공 기준

- 마피아 웹 흐름과 충돌 없이 별도 명령으로 라이어게임을 열 수 있다.
- 라이어게임 한 판이 Discord 안에서 완결된다.
- 신규 게임 추가 시 루트 아래 `/<new-game>` 모듈 패턴을 그대로 반복할 수 있다.
