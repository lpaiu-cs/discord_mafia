# 인게임 UI 예시

이 문서는 현재 구현 기준의 `웹 대시보드 인게임 UI` 를 예시로 정리한다.

- Discord는 로비, 참가 버튼, 입장 링크 발급만 담당한다.
- 실제 게임 진행은 `/game/:gameId` 웹 대시보드에서 이뤄진다.
- 아래 예시는 `모바일 우선` 기준이며, 데스크톱에서는 같은 정보를 더 넓은 그리드로 보여준다.

## 1. Discord 로비

공개 로비 메시지 예시:

```text
마피아42 시즌4 로비
규칙셋: 시즌4 밸런스

참가자 (5/8)
1. A (방장)
2. B
3. C
4. D
5. E
```

버튼 예시:

```text
[참가] [나가기] [시작]
```

참가 직후 ephemeral 예시:

```text
웹 대시보드 입장 링크를 발급했습니다.
[웹 대시보드 열기] [새 링크 발급]
```

## 2. 입장 / 재입장

- 입장 URL 은 `/auth/exchange?ticket=...`
- ticket 은 1회용 join ticket
- exchange 성공 시 세션 쿠키를 발급하고 `/game/:gameId` 로 이동
- 링크를 잃어버리면 `/mafia dashboard` 또는 `/mafia rejoin` 으로 새 링크를 다시 받는다

## 3. 모바일 공통 구조

모바일 기준 상단 헤더:

- 플레이어 닉네임
- 작은 `게임 ID`
- 역할 칩
- `252초 남음` 형태의 deadline 칩

모바일 기준 하단 고정 도크:

- `상태`
- `공개`
- `행동`
- `비밀`
- `로그`

도크는 화면 최하단에 고정되고, 현재 탭만 강조 표시된다.

## 4. 상태 탭

상태 탭은 현재 다음 순서로 렌더링된다.

### 내 정보 카드

- 직업
- 역할 설명
- 연인 정보가 있으면 추가 노출
- 사망 사유 / 성불 상태가 있으면 추가 노출
- 시민팀/마피아팀 여부는 카드 배경 톤으로 구분

### 미니 카드

- `Deadline`
- `행동 N개 가능`

### 좌석 그리드

- 1번부터 8번까지 고정 좌석
- 4칸 x 2줄 정사각형 카드
- 참가 순서 기준으로 채움
- 빈 자리는 `빈 자리`
- 현재 사용자는 `나` 배지
- 협박 상태는 `협박` 배지
- 성불 상태는 `성불` 배지
- 사망자는 카드 위에 붉은 X 오버레이

예시:

```text
[1 나 ] [2    ] [3    ] [4    ]
[민우  ] [하린 ] [서준 ] [지후 ]

[5 X  ] [6    ] [7    ] [8    ]
[소연  ] [도윤 ] [빈 자리] [빈 자리]
```

### 현재 대상

재판 단계면 현재 반론/찬반 대상 이름이 별도 카드로 노출된다.

## 5. 공개 탭

공개 탭은 공개 채팅 전용이다.

- 공개 메시지 목록
- 메시지 전송 폼
- 죽은 플레이어는 read-only
- 살아 있는 플레이어만 write 가능

메시지 예시:

```text
하린
저는 의심 가는 사람이 아직 없습니다.
20:11:03
```

## 6. 행동 탭

행동 탭은 현재 phase 와 역할 기준으로 가능한 행동만 보여준다.

예시:

- 밤 마피아: 대상 선택 셀렉트
- 밤 경찰: 조사 대상 선택
- 낮 기자: `기사 공개` 버튼
- 투표 단계: 투표 대상 선택
- 찬반 단계: `찬성 / 반대` 버튼
- 행동이 없으면 `현재 행동 없음`

행동은 모두 서버 authoritative 로 검증된다.

## 7. 비밀 탭

현재 세션으로 접근 가능한 비밀 채팅만 노출된다.

### 마피아 채팅

- 읽기: 마피아팀
- 쓰기: 밤

### 연인 채팅

- 읽기: 연인
- 쓰기: 밤

### 망자 채팅

- 읽기: 밤의 망자, 밤의 영매
- 쓰기: 밤의 비성불 망자, 밤의 영매

## 8. 로그 탭

로그 탭은 두 종류를 분리해서 보여준다.

- 공개 결과
- 개인 결과 / 시스템 로그

예시:

```text
공개 결과
기자 기사: 대상 님의 직업은 의사입니다.

개인 결과
조사 결과: 대상 님은 마피아가 아닙니다.
```

## 9. 단계별 체감 예시

### 밤

- 상태 탭 deadline 감소
- 행동 탭에서 밤 행동 제출
- 비밀 탭에서 마피아/연인/망자 채팅 접근

### 낮 토론

- 공개 탭에서 공개 채팅 가능
- 기자는 조건 충족 시 행동 탭에 공개 버튼 노출

### 투표

- 행동 탭에 투표 대상 셀렉트

### 최후의 반론

- 상태 탭에 현재 대상 노출
- 대상이 테러리스트면 별도 산화 선택 UI 가능

### 찬반 투표

- 행동 탭에 `찬성 / 반대`

## 10. 개발자 프리뷰 확인 포인트

`npm run dev:preview` 로 다음을 빠르게 점검할 수 있다.

- `/auth/exchange` -> 세션 쿠키 -> `/game/:gameId`
- 모바일 하단 도크
- 상태 탭 좌석 카드
- 죽은 좌석 X 오버레이
- 긴 닉네임 줄바꿈
- phase 별 행동 UI
- 공개/비밀 채팅 렌더링
